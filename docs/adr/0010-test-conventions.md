# ADR-0010: Test Naming and Tagging Conventions

- **Status:** accepted
- **Date:** 2025-01-15

## Context

ClinicSentry tests serve three audiences: developers iterating, CI gating merges, and regulators reviewing evidence. The same test suite must be (a) fast in the inner loop, (b) thorough in CI, and (c) traceable to specific regulatory requirements.

## Decision

### Naming

- File: `tests/test_<module>.py` mirroring the source package.
- Function: `test_<unit>_<behavior>_<condition>`, e.g., `test_chain_emit_assigns_monotonic_sequence_number`.
- One assertion concept per test; multiple `assert` statements only when they verify the same fact from different angles.

### Tagging (pytest markers)

| Marker | Use |
|--------|-----|
| `unit` | default — fast, pure, no I/O. |
| `integration` | crosses module boundaries, uses real backend (SQLite/Postgres). |
| `adapter` | requires a framework SDK; auto-skipped if not installed. |
| `slow` | > 1 second; excluded from inner loop, run in nightly CI. |
| `regulatory` | maps to a specific regulatory requirement via `pytestmark = pytest.mark.regulatory(rule="HIPAA-164.312-a-1")`. |
| `property` | hypothesis property test. |
| `mutation_target` | annotated for `mutmut` to prioritize. |

### Coverage

- Hard CI gate: ≥85% overall, ≥90% on novelty modules (propagation, confidence, chain, meddevice).
- Mutation score gate: ≥70% on novelty modules (run weekly, not per PR).

### Forbidden

- No `time.sleep` in tests; use freezegun or fake clocks.
- No real network calls; use `respx` / `aioresponses`.
- No PHI literals in test fixtures except in `tests/fixtures/synthetic_phi/`, generated by `tests/conftest.py::synthetic_phi_generator`.

## Consequences

- **Positive:** test suite is self-documenting; regulators can grep by marker; CI is fast.
- **Negative:** initial conformance pass on existing tests required.
- **Neutral:** the `regulatory` marker becomes a deliverable artifact for FDA/IEC submissions.
